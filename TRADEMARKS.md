# Toph Brand Guidelines

Toph is an open-source project. We want you to talk about it, share it, teach
it, review it, fork it, and build on top of it. These guidelines exist for one
reason: to avoid confusion about what is an official Toph release and what
isn't.

The Toph name, logo, and brand assets are not covered by the project's open
source license.

## Go ahead

Use the Toph name and logo when referring to the project — in blog posts,
videos, talks, tutorials, podcasts, package listings, compatibility notes,
screenshots, reviews, or anything else where you're talking about Toph
honestly. No permission needed.

A link back to the project repo or website is always appreciated but never
required.

## Just don't mislead

The one thing we ask is that you don't use the Toph name or logo in a way that
makes people think something is an official Toph release when it isn't. For
example:

- Don't present a modified fork as "Toph" without making it clear it's a fork.
- Don't use the Toph logo as your own product's branding.
- Don't imply official endorsement or partnership that doesn't exist.

If you're unsure whether your use might cause confusion, just reach out — we're
happy to chat.
